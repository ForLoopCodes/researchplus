This folder stores autonomous research memory files.

## Layout

- `ideas/` — one markdown file per idea or research direction.
- `evidence/` — short source notes, snippet extractions, and supporting passages.
- `checkpoints/` — compact live state for resume-after-compaction sessions.
- `queues/` — round-robin task queue and follow-up work.
- `templates/` — reusable markdown templates for new research artifacts.

## Working rules

- Keep idea notes short, numbered, and evidence-backed.
- Save the current live checkpoint in a single compact file.
- Keep the active queue in one file so the loop stays easy to resume.
- Use the templates when adding new notes so new sessions stay consistent.
